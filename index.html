Web demo for "Generative-based Singing Voice Separation using Denoising Diffusion Probabilistic Models"

Gen√≠s Plaja-Roglans, Yun-Ning Hung, Xavier Serra, Igor Pereira

This work was done during a research internship at Music.AI. Currently submitted to ICASSP 2025 and under review.
Please send the correspondence to: genis.plaja@upf.edu.

ABSTRACT: Music source separation is the process of estimating the individual elements in a music mixture, and is generally addressed by a neural network that masks or transforms an input mixture into the target source(s). However, despite the recent ground-breaking developments towards generative modelling for many audio problems, few generative-based source separation attempts have been made. In this study, we aim at exploiting the waveform generation ability of score-based diffusion models, which are not limited by a subtractive mask, to achieve high quality and leakage-free separation. We focus on the singing voice, an important source that has not been extracted from music mixtures using diffusion models yet. By directly generating waveforms, the complex phase is implicitly modeled. We strongly guide the generation process using musical mixes as conditioners and improve over existing generative separation systems, while achieving competitive objective scores when the model is trained with additional data. A diffusion separation toolkit is published.

What is this page about: This page includes audio examples and other interesting resources for our work on separating the singing voice from music mixtures using denoising diffusion probabilistic models (DDPM). We provide audio examples comparing our method with models in the literature but also comparing different configurations of our method.

diffdmx, a Python toolkit to run diffusion separation experiments: As claimed in the paper, we have created a Python installable package to run experiments with the diffusion models. Currently, the code is private but we plan to make it public after the review process. The idea behind the toolkit is that it contains all the building blocks to run several different diffusion experiments by only tweaking the configuration file. New modules and components can be easily added. See a code example below.

*please note all audio examples are randomly extracted from the MUSDB18HQ test set.

The diffdmx Python package

As mentioned before, we have created a Python package to run experiments with the diffusion models. Users can easily run diffusion separation experiments by only working on the Hydra configuration files. We provide files to reproduce the models in our work. See an example of how we could set up a model in the config files. 


            sample_rate: 44100
            model:
                diffusion:
                    in_channels: 2
                    out_channels: 2
                    dim: 1  ## Generating waveforms
                    channels: [16, 32, 64, 64, 128, 128, 256]
                    context_channels: [0, 0, 0, 256, 512, 512, 512]
                    factors: [1, 2, 4, 1, 4, 1, 4]
                    items: [1, 1, 2, 2, 2, 2, 2]
                    attentions: [0, 0, 0, 0, 1, 1, 1]
                    attention_heads: 8
                    attention_features: 64
                    inject_system: concatenation
                    sample_rate: ${sample_rate}
                conditioner:
                    _target_: diffdmx.conditioners.UniverseSeparator
                    generator_network: adp
                    attentions: [0, 0, 0, 0, 1, 1, 1]
                    bn_transformer: true
                    bn_size: 12
                    in_cond_channels: [64, 128, 256, 256, 512, 512, 512]
                    context_channels: ${model.diffusion.context_channels}
                    downsampling_factors: ${model.diffusion.factors}
                    sample_rate: ${sample_rate}
                    mel_channels: 80
                    loss_waveform: 'mse'
                    loss_bottleneck: 'mse'
                    loss_mel: null
                    auxiliary_skip_connections: true
                    mel_input: true
                optimizer:
                    lr: 1.e-6

Users may create their own conditioners or generator networks in the package code, and summon them using the config files. The package is designed to be modular and easy to extend.

Then, the code is executed through the command line: 


            python -m diffdmx.train +experiment=path/to/config/file
